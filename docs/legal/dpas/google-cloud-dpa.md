# Google Cloud Data Processing Addendum (CDPA)

**Source:** https://cloud.google.com/terms/data-processing-addendum
**Retrieved:** January 23, 2026
**Effective Date:** October 23, 2025
**Status:** Auto-accepted via Terms of Service

---

## Overview

The Cloud Data Processing Addendum describes the parties' obligations regarding processing and security of Customer Data under applicable privacy, data security, and data protection laws.

## Scope

Applies to:
- Google Cloud Platform services
- Google Workspace and Cloud Identity
- Looker (original)
- SecOps Services
- Mandiant Consulting Services and Managed Services
- Google Skills for Organizations
- Implementation Services

## Roles

- **Google:** Acts as Processor
- **Customer:** Acts as Controller or Processor

## Duration

Remains in effect until Google deletes all Customer Data.

## Data Processing

- Google processes data only per Customer's Instructions
- Instructions provided via Services, Admin Console, or written instructions

## Data Deletion

- Customer can delete data during Term (within 180 days)
- Upon Term end, Google deletes remaining Customer Data (within 180 days)

## Data Security

### Google's Security Measures
- Implements technical, organizational, physical measures
- Encryption of Customer Data
- Regular testing of effectiveness
- Updates permitted if not materially reducing security

### Data Incidents
- Prompt notification after becoming aware
- Details of incident, measures taken, contact point provided
- Notification within 48 hours for GDPR-related incidents

### Compliance Certifications
- ISO 27001
- ISO 27017
- ISO 27018
- SOC 2 and SOC 3 reports (updated annually)
- PCI DSS Attestation of Compliance

## Subprocessors

- Customer authorizes Google's engagement of disclosed Subprocessors
- 30-day notice before New Subprocessor engagement
- 90-day objection period (terminate for convenience)
- Subprocessor list: https://cloud.google.com/terms/subprocessors

## International Data Transfers

Compliant with:
- EU GDPR
- UK GDPR
- Swiss FADP
- CCPA
- LGPD (Brazil)

Transfer mechanisms:
- Standard Contractual Clauses (SCCs)
- UK International Data Transfer Agreement (IDTA)
- Alternative Transfer Solutions where available

## Data Subject Rights

Google enables Customer to:
- Access Customer Data
- Rectify data
- Restrict processing
- Export Customer Data

Google directs data subject requests to Customer.

## Data Center Security

### Infrastructure
- Geographically distributed data centers
- All production data in physically secure facilities
- Elimination of single points of failure
- Redundant dual circuits, switches, networks

### Power
- Redundant electrical systems
- UPS batteries for 10 minutes backup
- Backup generators for days of operation

### Data Transmission
- High-speed private links between data centers
- HTTPS/SSL/TLS encryption
- Ephemeral elliptic curve Diffie-Hellman (PFS methods)

### Access Control
- 24/7 security operations
- CCTV monitoring
- Electronic card key access
- Central authentication system
- Two-factor authentication
- Role-based access
- Audit trails

### Personnel Security
- Confidentiality requirements
- Background checks (legally permissible)
- Security training

## CCPA Compliance

Google will NOT:
- Sell or share Customer Personal Data
- Retain/use/disclose outside permitted purposes
- Combine with other information improperly
